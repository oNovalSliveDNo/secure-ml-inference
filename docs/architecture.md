# Архитектура прототипа secure-ml-inference

В документе описываются компоненты системы, их взаимодействие, потоки данных и логическое разделение обязанностей между клиентской и серверной сторонами.

## Общая схема

Прототип разделён на две логически независимые роли, соответствующие участникам протокола конфиденциального инференса:

- **Клиент** — владелец чувствительных табличных данных. Выполняет предобработку, кодирование, генерацию ключей, шифрование, расшифрование и финальное получение класса.
- **Сервер** — владелец заранее обученной ML-модели. Хранит только закодированные веса, получает зашифрованные признаки, вычисляет зашифрованный линейный score и возвращает его клиенту.

Физически обе роли реализованы как отдельные приложения, взаимодействующие по HTTP.

```
+-----------------------+          HTTP          +---------------------------+
|  Streamlit Client UI  | --------------------→ |  FastAPI Server           |
|                       |                       |                           |
|  - StandardScaler     |                       |  - Хранит w_int, b_int    |
|  - Fixed-point encode |                       |  - Принимает Enc(x)       |
|  - Paillier key gen   |                       |  - Вычисляет Enc(z)       |
|  - Encrypt features   |                       |  - Возвращает Enc(z)      |
|  - Decrypt score      |                       |                           |
|  - Sigmoid / Threshold|                       |                           |
+-----------------------+                       +---------------------------+
```

## Компоненты

### 1. Модуль `app/` — ядро бизнес-логики

Содержит логику, не зависящую от веб-фреймворков:

- **`config.py`** — конфигурационные параметры (scale, длина ключа, порог, пути и пр.)
- **`data.py`** — загрузка датасета, разделение на обучающую/тестовую выборки
- **`model.py`** — обучение модели, сохранение/загрузка, извлечение весов
- **`encoding.py`** — fixed-point кодирование и декодирование чисел
- **`crypto.py`** — генерация ключей Paillier, шифрование/дешифрование, сериализация
- **`client.py`** — класс `Client`, инкапсулирующий операции на стороне клиента
- **`server.py`** — класс `Server`, реализующий вычисление зашифрованного score
- **`inference.py`** — функции для четырёх режимов инференса
- **`metrics.py`** — утилиты расчёта метрик качества, времени и размера
- **`schemas.py`** — Pydantic-модели для API

### 2. `api/` — серверное FastAPI-приложение

- При старте загружает модель и веса с помощью функций `model.py`.
- Предоставляет конечные точки:
  - `GET /health` — проверка работоспособности
  - `GET /model/info` — метаинформация (размерность признаков, классы)
  - `POST /infer/encrypted` — основной защищённый эндпоинт
- Не имеет доступа к закрытому ключу или открытым признакам.

### 3. `ui/` — Streamlit-приложение клиента

Интерфейс реализован как **одна страница с тремя вкладками**:

- **«Демонстрация протокола»**:
  - выбор индекса тестового образца (`st.slider`);
  - запуск защищённого инференса (`st.button`);
  - просмотр этапов предобработки (масштабирование, fixed-point кодирование), зашифрованного вектора и HTTP-обмена с API;
  - сравнение `Baseline`/`Encoded`/`PHE` и отображение KPI (совпадение предсказаний, накладные расходы полезной нагрузки).
- **«Экспериментальные свидетельства»**:
  - просмотр CSV-таблиц из `results/tables/`;
  - просмотр PNG-графиков из `results/plots/`;
  - авто-комментарии по содержанию таблиц/графиков.
- **«Архитектура и модель угроз»**:
  - краткое текстовое описание потока защищённого инференса, границы доверия и модели угроз.

На клиенте локально выполняются: `StandardScaler`, fixed-point кодирование, генерация ключей Paillier, шифрование признаков, расшифрование результата и финальная классификация.

### 4. `experiments/` — скрипты воспроизводимых экспериментов

Одиннадцать пронумерованных скриптов, реализующих протокол экспериментов из `docs/experiment_protocol.md`. Каждый скрипт независим и сохраняет результаты в `results/`.

- **01–04: качество и корректность**
  - `01_train_baseline.py`
  - `02_validate_manual_inference.py`
  - `03_run_encoded_inference.py`
  - `04_run_phe_inference.py`
- **05–07: latency, payload, feature scaling**
  - `05_benchmark_latency.py`
  - `06_benchmark_payload.py`
  - `07_benchmark_feature_scaling.py`
- **08–11: dataset benchmark, key length, scale, API roundtrip**
  - `08_benchmark_datasets.py`
  - `09_benchmark_key_lengths.py`
  - `10_benchmark_scale.py`
  - `11_benchmark_api_roundtrip.py`

### 5. `results/` — генерируемые артефакты

- `models/` — обученная модель, scaler, веса
- `tables/` — CSV-файлы с метриками качества, времени, полезной нагрузки, масштабирования
- `plots/` — PNG-графики зависимостей времени и размера от числа признаков

## Поток данных в защищённом режиме

1. Клиент выбирает объект из тестового набора.
2. Применяет `StandardScaler` (предварительно загруженный при старте UI).
3. Преобразует стандартизованные признаки в целые числа с масштабным коэффициентом `SCALE`.
4. Генерирует ключевую пару Paillier (публичный и приватный ключи).
5. Шифрует целочисленный вектор публичным ключом.
6. Формирует JSON-запрос `EncryptedInferRequest` с полями:
   - `public_key_n: str`
   - `encrypted_features: list[str]`
   - `scale: int`
   - `feature_count: int`
7. Отправляет POST-запрос на `/infer/encrypted`.
8. Сервер (`POST /infer/encrypted`):
   - Валидирует согласованность `feature_count`, длины `encrypted_features` и размерности модели.
   - Валидирует совпадение `scale` с серверной конфигурацией.
   - Восстанавливает публичный ключ из `public_key_n` и десериализует шифртексты.
   - Вычисляет зашифрованный score: `Enc(z_int) = Σ Enc(x_i)*w_int_i + Enc(b_int)`.
   - Возвращает `EncryptedInferResponse` с полями:
     - `encrypted_score: str`
     - `server_compute_ms: float`
     - `feature_count: int`
9. Клиент:
   - Расшифровывает `score_int` приватным ключом.
   - Переводит в вещественное число: `z = score_int / (SCALE * SCALE)`.
   - Вычисляет сигмоиду: `p = 1 / (1 + exp(-z))`.
   - Применяет порог `0.5` для получения класса.
   - Сравнивает с baseline-предсказанием.

## Математика линейной части

Для одного вектора стандартизованных признаков `x` (после масштабирования):

```
z = w₁x₁ + w₂x₂ + … + wₙxₙ + b
p = sigmoid(z)
y = 1 if p ≥ 0.5 else 0
```

Сервер вычисляет только `Enc(z)`, не имея доступа к `x` в открытом виде.

## Масштабирование и кодирование

- Все значения умножаются на `SCALE` (по умолчанию 10 000) и округляются до целых.
- Веса и смещение кодируются один раз при обучении и хранятся на сервере.
- Клиент кодирует каждый входной вектор непосредственно перед шифрованием.
- После расшифрования результат делится на `SCALE²` для восстановления исходного масштаба.

## Stateless и конфиденциальность

- Эндпоинт `/infer/encrypted` является **stateless на уровне запроса**: каждое обращение полностью самодостаточно и содержит открытый ключ и вектор шифртекстов; межзапросное пользовательское состояние не накапливается.
- На старте приложения сервер загружает только параметры модели в fixed-point (`w_int`, `b_int`) и метаданные (`feature_count`, `classes`). Эти данные неизменны между запросами и не содержат пользовательский plaintext.
- Сервер не получает и не хранит:
  - открытые признаки пользователя;
  - закрытый ключ Paillier;
  - расшифрованный линейный score или итоговую вероятность/класс.
- Передача plaintext-признаков на сервер отсутствует: по сети отправляются только `public_key_n` и `encrypted_features` (плюс служебные поля `scale`, `feature_count`).

## Допущения

- Параметры `StandardScaler` доступны клиенту (не являются секретом).
- Ключи генерируются клиентом для каждой сессии или один раз (определяется параметрами).
- Сетевое взаимодействие происходит в доверенной локальной среде (для демонстрации).

## Контейнерный запуск

Для локального развёртывания используются два контейнера в `docker-compose.yml`:
- `api`: FastAPI-сервер на `:8000`;
- `ui`: Streamlit-клиент на `:8501`, зависит от `api`.

Быстрая проверка после старта `docker compose up --build`:
- `GET http://localhost:8000/health` возвращает `{"status":"ok"}`;
- интерфейс доступен на `http://localhost:8501`.
