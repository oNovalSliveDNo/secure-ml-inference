# Алгоритм Paillier (учебный конспект)

> Этот материал носит учебный характер и не является руководством по production-криптографии.

## 1. Генерация ключей

Пусть выбираются два больших простых числа `p` и `q`.

1. Вычисляем модуль:
   - `n = p * q`
   - `n^2` используется как рабочий модуль для шифртекстов.
2. Вычисляем функцию Кармайкла (в базовой форме для Paillier):
   - `λ = lcm(p - 1, q - 1)`
3. Выбираем `g ∈ Z*_{n^2}` (часто берут `g = n + 1` в учебных и практических схемах).
4. Вычисляем:
   - `μ = (L(g^λ mod n^2))^{-1} mod n`
   - где `L(u) = (u - 1) / n`

Обратный элемент существует при корректном выборе параметров (`g`, `p`, `q`).

## 2. Структура ключей

- **Публичный ключ**: `(n, g)`
- **Приватный ключ**: `(λ, μ)`

## 3. Формулы шифрования и расшифрования

Пусть сообщение `m` принадлежит `Z_n`.

### Шифрование

`c = g^m * r^n mod n^2`, где `r ∈ Z_n*`.

### Расшифрование

`m = L(c^λ mod n^2) * μ mod n`

## 4. Гомоморфные свойства

### Гомоморфное сложение

Для `c1 = Enc(m1)` и `c2 = Enc(m2)`:

`c_add = c1 * c2 mod n^2`

Тогда после расшифрования:

`Dec(c_add) = (m1 + m2) mod n`

### Умножение на скаляр

Для `c = Enc(m)` и целого `k`:

`c_mul = c^k mod n^2`

Тогда:

`Dec(c_mul) = (k * m) mod n`

## 5. Ограничения и практические замечания

- Сообщение должно удовлетворять ограничению: `0 <= m < n`.
- Случайный параметр `r` должен выбираться из `Z_n*`, то есть `gcd(r, n) = 1`.
- Повторное использование одного и того же `r` для разных сообщений недопустимо в реальных системах.
- Примеры с маленькими простыми числами подходят **только для обучения**.
