# Протокол экспериментальной оценки

В документе описываются цели, методика, сценарии и критерии успешности экспериментов, проводимых в рамках проекта `secure-ml-inference`.

## Цели экспериментов

1. Подтвердить, что предсказания защищённого PHE-инференса статистически идентичны предсказаниям открытой модели.
2. Количественно оценить издержки, вносимые шифрованием:
   - Дополнительное время на клиенте и сервере;
   - Увеличение объёма передаваемых данных.
3. Исследовать влияние количества признаков на временные и объёмные характеристики защищённого протокола.

## Общие параметры

| Параметр               | Значение                                 |
|------------------------|------------------------------------------|
| Датасет                | Breast Cancer Wisconsin Diagnostic       |
| Размер тестовой выборки | 20% от всего датасета (stratified)     |
| Random state           | 42                                       |
| Модель                 | StandardScaler + LogisticRegression      |
| Solver                 | lbfgs, max_iter=1000                     |
| Fixed-point scale      | 10 000 (по умолчанию)                    |
| Длина ключа Paillier   | 1024 (основной режим)                    |
| Числа признаков для масштабирования | 5, 10, 20, 30 (top-k по |w|) |
| Количество повторений для бенчмарков | 30 (или 50)                  |
| Измерение времени      | `time.perf_counter()`                    |

## Режимы инференса (4 сценария)

### 1. Plaintext baseline
Обычный вызов `model.predict()` / `model.predict_proba()` на тестовой выборке.  
Служит эталоном для сравнения.

### 2. Manual plaintext inference
Ручной расчёт `z = X_scaled @ w + b` с последующей сигмоидой и порогом.  
Проверяет, что математика совпадает с sklearn.

### 3. Encoded plaintext inference (без шифрования)
Имитация целочисленной арифметики фиксированной точки:
- Кодирование признаков и весов в целые числа.
- Вычисление `score_int` в целых числах.
- Обратное преобразование в вещественное `z`.
Этот режим оценивает потерю точности только от округления.

### 4. PHE inference (со сквозным шифрованием)
Полноценный защищённый протокол, описанный в `architecture.md`.  
Выполняется на тестовой выборке (или на подмножестве, если время неприемлемо большое).

## Метрики качества

Для каждого режима (кроме manual) рассчитываются:

- Accuracy
- Precision
- Recall
- F1-score
- ROC-AUC
- **Match rate** с baseline (доля совпадений предсказаний)
- **Mean absolute score error** относительно baseline-прогноз вероятностей

Метрики записываются в `results/tables/quality_metrics.csv` (одна строка на режим).

## Эксперимент 1: Обучение baseline

Скрипт `01_train_baseline.py`:

- Загружает датасет, делит на train/test.
- Обучает пайплайн, сохраняет модель, scaler, веса и метаданные.
- Вычисляет метрики качества открытой модели и записывает в CSV.

## Эксперимент 2: Проверка manual plaintext

Скрипт `02_validate_manual_inference.py`:

- Для всей тестовой выборки вычисляет `z` через веса, сигмоиду и класс.
- Сравнивает с результатами `model.predict()` и `model.predict_proba()`.
- Выводит долю совпадений и максимальную абсолютную ошибку score.
- Критерий успешности: совпадение 100% (или отличие ≤1e-12 из-за floating-point).

## Эксперимент 3: Encoded plaintext

Скрипт `03_run_encoded_inference.py`:

- Применяет fixed-point кодирование ко всем признакам тестового набора.
- Вычисляет integer score, декодирует, получает предсказания.
- Сравнивает с baseline, вычисляет все метрики качества.
- Сохраняет строку "Encoded plaintext" в quality_metrics.csv.

## Эксперимент 4: PHE inference

Скрипт `04_run_phe_inference.py`:

- Для каждого объекта тестового набора (или подвыборке, если тест большой):
  - Клиент: масштабирует, кодирует, шифрует.
  - Сервер (локальный объект `Server`) вычисляет зашифрованный score.
  - Клиент: расшифровывает, получает класс.
- Собирает предсказания, сравнивает с baseline и с encoded plaintext.
- Записывает строку "PHE inference" в quality_metrics.csv.

## Эксперимент 5: Замеры времени (Latency benchmark)

Скрипт `05_benchmark_latency.py`:

- Многократно замеряет компоненты времени для одного случайного объекта:
  - Preprocessing time (scaler)
  - Encoding time (float → int)
  - Key generation time
  - Encryption time (30 признаков)
  - Server computation time (encrypted linear score)
  - Decryption time
  - Sigmoid + threshold time
  - Total round-trip time
- Результаты (среднее, std, медиана) записываются в `latency_metrics.csv`.

## Эксперимент 6: Объём передаваемых данных (Payload benchmark)

Скрипт `06_benchmark_payload.py`:

- Сравнивает размер гипотетического plaintext-запроса (JSON-список float) и реального encrypted-запроса (список строк-шифртекстов).
- Аналогично для ответа.
- Вычисляет коэффициенты роста (`request_growth_factor`, `response_growth_factor`).
- Записывает таблицу в `payload_metrics.csv`.

## Эксперимент 7: Влияние числа признаков (Feature scaling)

Скрипт `07_benchmark_feature_scaling.py`:

- Выбирает `k` признаков из топ-`k` по абсолютной величине весов модели (наиболее значимых).
- Для каждого `k ∈ {5, 10, 20, 30}` измеряет:
  - Время шифрования
  - Время серверного вычисления
  - Время расшифрования
  - Общее время инференса
  - Размер зашифрованного запроса
- Сохраняет таблицу `feature_scaling_metrics.csv`.
- Генерирует три графика в `results/plots/`:
  1. `feature_count_vs_total_time.png`
  2. `feature_count_vs_request_size.png`
  3. `feature_count_vs_server_time.png`

## Оборудование и программное окружение

Все эксперименты проводятся в одинаковом программном окружении (Docker-контейнер), чтобы обеспечить воспроизводимость. Характеристики хостовой машины (CPU, RAM) фиксируются в отчёте. Для замеров используются только CPU-операции (без GPU).

## Критерии успешности

- Качество модели на PHE совпадает с baseline (match rate ≥ 99%).
- Максимальная абсолютная ошибка score не превышает допустимого порога, обусловленного только квантованием.
- Получены количественные оценки роста времени и размера, которые можно обсудить в выводе.
- Все скрипты выполняются без ошибок и воспроизводят результаты при повторном запуске.
